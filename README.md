# Mimir — The Well of Knowledge

> *In Norse myth, Mimir guards the well beneath Yggdrasil — the source of all wisdom.
> Odin sacrificed his eye just to drink from it.*
>
> *This is my attempt to build something worthy of that name.*

---

Mimir is a personal desktop application I am building for myself as a student tired of scattered notes, forgotten flashcards, and the creeping anxiety of not knowing what to study next. It is a local AI study companion — one that lives entirely on your machine, knows your weak spots, remembers what you have studied, and pushes back when you are slacking.

Everything runs locally. No API keys. No data leaving your machine. The LLM is served by Ollama, the conversation history lives in SQLite, and semantic memory is stored in ChromaDB. If you unplug the internet, Mimir still works.

## Install

**Prerequisites:**

1. **Ollama** — [ollama.com/download](https://ollama.com/download)
2. **Pull the model** (~6 GB — do this first):
   ```
   ollama pull qwen3.5:9b
   ```

**Then download and run the installer from the [Releases page](https://github.com/Sarthak-47/Mimir/releases).**

The installer includes the Python backend — no Python, Rust, or Node.js required.

> Windows x64 only for now.

## What it does

Mimir is built around a conversational interface — you talk to it like a tutor, and it responds like one. Behind that interface is a ReAct agent loop that decides when to explain, when to quiz, when to pull up your past sessions, and when to tell you which topics you are weakest in and should revisit tonight.

**What works:**

- Register and log in — JWT auth with bcrypt passwords, token stored for the session
- Chat with the local LLM through a WebSocket-streamed interface; response tokens appear as they arrive
- Norse-themed thinking indicator while the model reasons ("Consulting the Well of Urd…", "The Norns weave their answer…")
- The ReAct agent loop selects tools based on your message — explain, quiz, flashcards, schedule, recall
- Quizzes generated by the agent render as inline interactive cards in the chat; correct/wrong state per option
- Flashcard decks appear in-chat with flip interaction and navigation between cards
- Trials view — dedicated quiz runner with subject/topic selector and full score card on completion
- Reckoning dashboard — stats (days, accuracy, streak, total trials), per-topic confidence bars, quiz history
- Chronicle — full paginated conversation history in the same bubble style as the live chat
- Scrolls — file library showing uploaded documents with processed/pending indexing status
- PDF and image uploads: text is extracted (PyMuPDF for PDFs, Tesseract for images), chunked at 512 characters with overlap, and indexed into ChromaDB so Mimir can recall the content
- Progress stats and weak topic bars in the right panel are fetched live from the backend
- Spaced repetition logic updates review dates after every quiz submission (4h / 1d / 3d / 7d intervals)
- Exam date tracking — set your exam date in the sidebar, the countdown in the right panel is real
- Disciplines (subjects) are created and deleted live via the API, not hardcoded
- APScheduler runs hourly overdue-topic checks and a daily streak recompute in the background
- The full three-column layout — navigation, chat, stats panel — is in place with the Norse aesthetic

**What remains:**

- Exam date → personalised revision schedule (the agent has the `schedule` tool; the UI to trigger it deliberately is not surfaced yet)
- Notification system for overdue review reminders (scheduler detects them, nowhere to send them yet)
- Mac and Linux builds

## Tech stack

| Layer | Technology |
|-------|-----------|
| Desktop shell | Tauri (Rust) — produces a ~10 MB native binary |
| Frontend | React 18 with TypeScript, bundled by Vite |
| Styling | CSS custom properties, no component library |
| Typography | Cinzel and Crimson Text — loaded from Google Fonts |
| Backend | FastAPI with async SQLAlchemy |
| Local LLM | Ollama running qwen3.5:9b |
| Vector memory | ChromaDB, embedded and persistent |
| Relational data | SQLite |
| File parsing | PyMuPDF for PDFs, Tesseract for OCR on images |
| Scheduling | APScheduler AsyncIOScheduler |

The UI aesthetic is deliberately borrowed from God of War — forest green and gold on near-black stone backgrounds, angular clip-paths instead of rounded corners, Elder Futhark runes as navigation icons. It is a study app that takes itself seriously.

## Running from source

You need Rust, Python 3.11, Node.js, and Ollama installed. The backend must be running before the frontend — a boot splash polls `/health` on startup and waits automatically.

```bash
# Pull the model (~6 GB — do this first)
ollama pull qwen3.5:9b

# Backend
cd backend
py -3.11 -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --port 8000

# Frontend (separate terminal)
cd frontend
npm install
npm run dev

# Tauri desktop window (separate terminal, from repo root)
cargo tauri dev
```

To build the full installer locally:

```bash
# 1. Build the Python backend bundle (one-time, ~197 MB output)
cd backend
pyinstaller mimir-backend.spec

# 2. Copy bundle into Tauri's resource directory
xcopy /E /I dist\mimir-backend ..\src-tauri\binaries\mimir-backend

# 3. Build the NSIS installer
cd ..
cargo tauri build --bundles nsis
# Output: src-tauri/target/release/bundle/nsis/Mimir_0.1.0_x64-setup.exe
```

## Design notes

The visual language is intentionally severe. No rounded corners. No pastel gradients. Panels are separated by single-pixel borders and thin gold engraving lines. Navigation uses Elder Futhark runes as icons. Typography is set in Cinzel for anything structural and Crimson Text for anything you read. The colour palette is forest green on stone black with gold as the only accent.

It is the kind of UI that should feel like opening a grimoire, not launching a SaaS dashboard.
