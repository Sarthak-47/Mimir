# Mimir — The Well of Knowledge

> *In Norse myth, Mimir guards the well beneath Yggdrasil — the source of all wisdom.
> Odin sacrificed his eye just to drink from it.*
>
> *This is my attempt to build something worthy of that name.*

---

Mimir is a personal desktop application I am building for myself as a student tired of scattered notes, forgotten flashcards, and the creeping anxiety of not knowing what to study next. It is a local AI study companion — one that lives entirely on your machine, knows your weak spots, remembers what you have studied, and pushes back when you are slacking.

It is not a polished product. It is not backed by a company. It is a solo project, written in spare hours between exams, and it is very much still a work in progress. The architecture is laid out, the core agent loop is running, and the UI exists — but there are rough edges, missing features, and whole sections that are stubbed. That is the honest state of things.

I am building it because I want to use it, and I am sharing it because someone else might find it useful too.

---

## What it does

Mimir is built around a conversational interface — you talk to it like a tutor, and it responds like one. Behind that interface is a ReAct agent loop that decides when to explain, when to quiz, when to pull up your past sessions, and when to tell you which topics you are weakest in and should revisit tonight.

Everything runs locally. No API keys. No data leaving your machine. The LLM is served by Ollama, the conversation history lives in SQLite, and semantic memory is stored in ChromaDB. If you unplug the internet, Mimir still works.

**What works right now:**

- Chat with the local LLM (qwen2.5:14b via Ollama) through a WebSocket-streamed interface
- The ReAct agent selects tools based on your message and acts on them
- Quizzes generated by the agent render as inline interactive cards in the chat
- Flashcard decks appear in-chat with flip interaction
- PDF and image uploads are sent to the backend for parsing and summarisation
- Progress stats and weak topic bars in the right panel are fetched live from the backend
- Spaced repetition logic updates review dates after every quiz submission
- The full three-column layout — navigation, chat, and the Warrior's Record panel — is in place

**What is still being built:**

- The Trials view (dedicated quiz mode outside the chat)
- The Reckoning dashboard (full progress charts and topic history)
- The Chronicle view (browseable conversation history)
- The Scrolls library (managing uploaded files)
- Multi-user authentication fully wired end-to-end
- Exam date tracking and personalised revision schedule generation
- The production Tauri binary builds and packaging

## Tech stack

The stack was chosen to keep everything local, fast to iterate on, and genuinely lightweight once bundled.

| Layer | Technology |
|-------|-----------|
| Desktop shell | Tauri (Rust) — produces a ~10MB native binary |
| Frontend | React 18 with TypeScript, bundled by Vite |
| Styling | CSS custom properties, no component library |
| Typography | Cinzel and Crimson Text — loaded from Google Fonts |
| Backend | FastAPI with async SQLAlchemy |
| Local LLM | Ollama running qwen2.5:14b |
| Vector memory | ChromaDB, embedded and persistent |
| Relational data | SQLite |
| File parsing | PyMuPDF for PDFs, Tesseract for OCR on images |

The UI aesthetic is deliberately borrowed from God of War — forest green and gold on near-black stone backgrounds, angular clip-paths instead of rounded corners, Elder Futhark runes as navigation icons. It is a study app that takes itself seriously.

## Running it locally

Full prerequisites and step-by-step installation instructions are in [INSTALL.md](./INSTALL.md). The short version:

You need Rust, Python 3.11, Node.js, and Ollama installed. Then:

```bash
# Pull the model (about 9 GB — do this first while you set up the rest)
ollama pull qwen2.5:14b

# Backend
cd backend
py -3.11 -m venv .venv
.venv\Scripts\activate        # or: source .venv/bin/activate on Mac/Linux
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Frontend (separate terminal)
cd frontend
npm install
npm run dev

# Tauri desktop window (separate terminal, from repo root)
cargo tauri dev
```

The app window opens automatically once all three are running.

## Design notes

The visual language is intentionally severe. No rounded corners. No pastel gradients. Panels are separated by single-pixel borders and thin gold engraving lines. Navigation uses Elder Futhark runes as icons. Typography is set in Cinzel for anything structural and Crimson Text for anything you read. The colour palette is forest green on stone black with gold as the only accent.

It is the kind of UI that should feel like opening a grimoire, not launching a SaaS dashboard.
